## 2. Actividades Prácticas

### Actividad 1.1: Configuración de Git y Repositorio

**Entregable:** Creación del repositorio local y remoto en GitHub con la estructura de directorios solicitada (`Semana1/Consolidado/` y `Semana1/Actividad1/`) para la gestión del control de versiones mediante *commits* documentados.

### Actividad 1.2: Entorno de Trabajo

**Entregable:** Instalación y validación del entorno de programación en Python mediante Jupyter Notebook. Se comprobó la correcta instalación de las librerías fundamentales para ciencia de datos: `NumPy`, `Pandas`, `Matplotlib`, `Seaborn` y `Scikit-learn`.

### Actividad 1.3: Casos de Uso en la Industria

**Entregable:** Documentación sobre cómo empresas líderes aplican la ciencia de datos:

- **Netflix:** Uso de filtrado colaborativo y A/B testing para retención de usuarios.
- **Amazon:** Análisis de reglas de asociación y series temporales para optimizar la cadena de suministro y ventas cruzadas.
- **Spotify:** Procesamiento de lenguaje natural y análisis de audio crudo para el descubrimiento semanal de música.

### Actividad 1.4: Exploración de Fuentes de Datos (Kaggle)

**Entregable:** Análisis exploratorio de la plataforma Kaggle y selección del dataset *"Netflix Movies and TV Shows"*. Se identificó la estructura de los datos (tabulares y categóricos) y se plantearon preguntas de negocio, como la proporción de películas vs. series y la evolución del catálogo por año y país de origen.

## 3. Resumen de Aprendizaje

- Comprendí la intersección de habilidades necesarias para la ciencia de datos (programación, matemáticas y conocimiento del negocio) y los roles específicos como *Data Engineer*, *Data Analyst* y *Data Scientist*.
- Aprendí a identificar y aplicar las 5 "V" del Big Data (Volumen, Velocidad, Variedad, Veracidad y Valor) en un escenario real de negocio electrónico (DeportivaMX).
- Analicé la diferencia entre bases de datos relacionales y NoSQL, entendiendo por qué una base de datos orientada a documentos como MongoDB (almacenando datos en JSON) es ideal para catálogos de productos flexibles y escalables en arquitecturas de nube (Data Lakehouse).
- Reforcé la lógica computacional básica y las herramientas de control de versiones (Git/GitHub), sentando las bases prácticas para el desarrollo del curso.
